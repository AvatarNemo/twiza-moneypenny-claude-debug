Il Gateway

Il cuore di TWIZA Moneypenny: il servizio che orchestra AI, canali e strumenti.

🧠 Cos'è il Gateway

Il Gateway è il processo centrale di OpenClaw. Gira sulla porta 18789 e gestisce: sessioni di chat, connessioni ai canali, chiamate ai modelli AI, esecuzione di tool e strumenti, autenticazione e sicurezza. Tutto passa dal Gateway.

⚡ Comandi Essenziali

Stato del Gateway

openclaw gateway status

Avvio / Arresto / Riavvio

openclaw gateway start openclaw gateway stop openclaw gateway restart

Configurazione

openclaw config get # mostra tutta la config openclaw config get agents.model # mostra un campo specifico openclaw config set agents.model anthropic/claude-sonnet-4-20250514

Dashboard

openclaw dashboard # apre la dashboard nel browser

Log

openclaw gateway logs # log in tempo reale openclaw gateway logs --lines 100 # ultime 100 righe

🔐 Sicurezza

Autenticazione

Il Gateway usa token di autenticazione. In modalità locale (loopback), l'accesso è automatico. Per accesso remoto, serve un token configurato.

openclaw config set gateway.auth.token YOUR_SECRET_TOKEN

Binding di Rete

Di default il Gateway ascolta solo su localhost (127.0.0.1). Per accesso da rete:

openclaw config set gateway.bind 0.0.0.0 # tutte le interfacce openclaw config set gateway.bind loopback # solo localhost (default)
⚠️ Esporre il Gateway su 0.0.0.0 senza token è un rischio di sicurezza. Configura SEMPRE un token prima.

Approvazione Comandi

I comandi di sistema eseguiti dall'AI richiedono approvazione. Puoi configurare whitelist e policy per automatizzare quelli sicuri.
