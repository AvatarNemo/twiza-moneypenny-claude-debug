🔧 Risoluzione Problemi

Problemi comuni e come risolverli. Clicca su un problema per espandere la soluzione.

❌ Il Gateway non si avvia +

Cause comuni: Porta 18789 già occupata, Node.js troppo vecchio, errore di sintassi nella configurazione.

✅ Soluzione:

# Controlla se la porta è occupata lsof -i :18789 # Verifica versione Node.js (serve 22.14+ o 24) node --version # Valida la configurazione openclaw config get # Controlla i log per errori specifici openclaw gateway logs --lines 50
📱 WhatsApp si disconnette +

Cause comuni: Sessione scaduta, WhatsApp Web aperto altrove, aggiornamento app telefono.

✅ Soluzione:

# Ri-effettua il pairing WhatsApp # Vai nella dashboard → Canali → WhatsApp → Ri-connetti # Oppure riavvia il gateway per forzare un nuovo QR openclaw gateway restart # Se persiste, cancella la sessione e ricollega rm -rf ~/.openclaw/state/whatsapp/signalstore openclaw gateway restart
🤖 Errori del modello AI +

Cause comuni: API key non valida, provider down, rate limit raggiunto, modello non disponibile.

✅ Soluzione:

# Verifica la configurazione del modello openclaw config get agents.model # Controlla la API key openclaw config get providers # Testa con un modello diverso openclaw config set agents.model anthropic/claude-sonnet-4-20250514 # Per errori 429 (rate limit): aspetta qualche minuto e riprova # Per Ollama locale: verifica che sia in esecuzione ollama list ollama serve
🐌 Risposte lente +

Cause comuni: Modello troppo grande per l'hardware, context overflow, troppi tool in uso, rete lenta.

✅ Soluzione:

# Usa un modello più piccolo/veloce openclaw config set agents.model groq/llama-3.3-70b-versatile # cloud veloce openclaw config set agents.model ollama/qwen2.5:1.5b # locale piccolo # Controlla lo status per vedere l'uso di token # Nella webchat: /status # Riduci la finestra di contesto se necessario # Compatta la conversazione con /compact
🚫 Permesso negato (exec) +

Cause comuni: Comando non approvato, policy di sicurezza restrittiva.

✅ Soluzione: Quando l'AI chiede di eseguire un comando, la dashboard mostra un prompt di approvazione. Puoi approvare una volta (/approve allow-once) o sempre per quel comando (/approve allow-always). Per configurare la policy:

# Vedi le approvazioni attuali cat ~/.openclaw/exec-approvals.json # Configura la security policy openclaw config get agents.tools.exec
💾 Problemi di memoria/contesto +

Cause comuni: Conversazione troppo lunga, file grandi nel contesto, troppi tool result.

✅ Soluzione:

# Compatta la conversazione nella webchat /compact # Inizia una nuova sessione /new # Controlla l'uso attuale /status
🌐 WebChat non si carica +

Cause comuni: Gateway non avviato, porta bloccata, problema CORS, firewall.

✅ Soluzione:

# Verifica che il gateway sia attivo openclaw gateway status # Apri la dashboard (auto-rileva la porta) openclaw dashboard # Se accedi da un'altra macchina, verifica il binding openclaw config get gateway.bind # Deve essere 0.0.0.0 per accesso non-locale
🔄 Problemi con gli aggiornamenti +

Cause comuni: Versione npm incompatibile, cache corrotta, permessi file.

✅ Soluzione:

# Aggiornamento standard openclaw update # Se fallisce, aggiornamento manuale npm install -g openclaw@latest # Verifica la versione openclaw --version # Dopo l'aggiornamento, riavvia il gateway openclaw gateway restart

📋 Comandi Utili per il Debug

openclaw status — Stato completo del sistema
openclaw gateway status — Stato del gateway
openclaw gateway logs — Log in tempo reale
openclaw config get — Mostra configurazione completa
openclaw dashboard — Apri dashboard nel browser
openclaw --version — Versione installata
openclaw gateway restart — Riavvia il gateway
